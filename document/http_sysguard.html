ngx_http_sysguard_module

This module can guard the system load. When the swap memory or the system load steps over the configure guard line, all the requests will be redirected to specific url. To enable this feature, you should add the configure argument
--with-http_sysguard_module 

    server {
        sysguard on;

        sysguard_load load=1.1 action=/loadlimit;
        sysguard_mem swapratio=90% action=/swaplimit;

        location /loadlimit {
            return 500;
        }

        location /swaplimit {
            return 500;
        }
    }
Note, this module needs the system function of sysinfo() or getloadavg(). If your system can't supply any of these functions, this module can't be enabled.

Directive
Syntax: sysguard [on | off]
Default: sysguard off
Context: http, server

Turn on or off this module

Syntax: sysguard_load load=number [action=/url]
Default: none
Context: http, server

Specify the guard load to protect your system. When the system load exceeds the guard load in one mimute, all the requests will be redirected to the specific url. If the action is absent, return 503.

Syntax: sysguard_mem swapratio=ratio% [action=/url]
Default: none
Context: http, server

Specify the guard swap memory ratio to protect your system. "swapratio" means the used percentage of swap memory. If current system used ratio is more than the guard ratio, all the requests will be redirected to specific url. If the action url is absent, return 503. If you disable your swap memory, this directive is useless.

Syntax: sysguard_interval time
Default: sysguard_interval 1s
Context: http, server

Specify the time interval to fetch your system infomation. The default value is 1 second. It means it just calls this function in 1 second a time.

Syntax: sysguard_log_level [info | notice | warn | error]
Default: sysguard_log_level error
Context: http, server

This directive specify the log level when the protecting action triggers.
