ngx_http_limit_req_module
Directive
Syntax: limit_req_log_level info | notice | warn | error
Default: limit_req_log_level warn
Context: http

The same as the Nginx's limit_req_log_level

Syntax: limit_req_zone $session_variable1 $session_variable2 ... zone=name_of_zone:size rate=rate
Default: none
Context: http

It's similar to the Nginx, while Tengine can support several limit variables and limit_req_zone directives in one block. For example: 

    limit_req_zone $binary_remote_addr zone=one:3m rate=1r/s;
    limit_req_zone $binary_remote_addr $uri zone=two:3m rate=1r/s;
    limit_req_zone $binary_remote_addr $request_uri zone=thre:3m rate=1r/s;
The above second directive means the request from the same remote ip and uri will be limited to one request per second.

Syntax: limit_req [on | off] | zone=zone_name burst=burst [forbid_action=action] [nodelay]
Default: none
Context: http, server, location

The parameter of 'zone', 'burst' and 'nodelay' are the same meaning as the original limit_req moduel.

You can turn this directive on or off, default is on. This module support multiple directives in a block. If there are multiple limit_req directives, all the limit directives are executed in order. If the first limitation is exceeded, the first limit action will be triggered.

forbid_action means the action to do when some limit conditions are triggered. It can be the name of location, default action is 503. For example: 

    limit_req_zone $binary_remote_addr zone=one:3m rate=1r/s;
    limit_req_zone $binary_remote_addr $uri zone=two:3m rate=1r/s;
    limit_req_zone $binary_remote_addr $request_uri zone=three:3m rate=1r/s;

    location / {
        limit_req zone=one burst=5;
        limit_req zone=two forbid_action=@test1;
        limit_req zone=three burst=3 forbid_action=@test2;
    }

    location /off {
        limit_req off;
    }

    location @test1 {
        rewrite ^ /test1.html;
    }

    location @test2 {
        rewrite ^  /test2.html;
    }

Syntax: limit_req_whitelist geo_var_name=var_name geo_var_value=var_value
Default: none
Context: http, server, location

It means the whitelist. This directive needs work with geo module, the 'geo_var_name' is the variable name which is specified in geo module, 'geo_var_value' is the variable value. For example: 

    geo $white_ip {
        ranges;
        default 0;
        127.0.0.1-127.0.0.255 1;
    }

    limit_req_whitelist geo_var_name=white_ip geo_var_value=1;
It means the ip from the range of 127.0.0.1 to 127.0.0.255 will be ignored by the 'limit_req' directive.
